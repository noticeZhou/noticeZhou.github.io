JavaScript函数参数传递

  • JavaScript

大家都知道js参数传递是按值传递，传递参数的时候就直接将实参复制给形参就可以了。但是，我们知道js有两种数据类型，分别为基本类型和引用类型。两种类型的复制是不一样的，我们先来看一下复制。
基本类型的值就是保存在内存中，复制很简单，就是直接将变量的值copy到新变量对应的内存空间中就ok了。但是，引用类型有些不一样，因为引用类型存储在内存中的值只是一个指针，指向引用类型，而引用类型本身是保存在堆中的。所以只是一个对象的变量名赋值给一个新的变量的拷贝，叫做浅拷贝，只是把对象的指针复制给了新变量。这样造成的后果就是，两者的改变是相关的，只要有一个变了，另一个跟着就会变，我们来看一个栗子： 

var obj1 = {
    "name": "andy",
    "age": 21
}
var obj2 = obj1;
obj2.name = "rose" //改变obj2
console.log(obj1.name); //rose

从例子中可以很清楚的看到两个对象之间的关联。

现在回到我们的正题上，参数传递。先看下面一段代码

function setName(obj) {
    obj.name = "rose"
}
var person = {};
setName(person);
console.log(person.name); //rose

不是说js是按值传递参数的吗？那为什么函数内部的变化会影响到外部局部变量呢？又欺负我年轻。别急，再看一段代码

function setName(obj) {
    obj.name = "rose";
    obj = new Object();
    obj.name = "andy"
}
var person = {};
setName(person);
console.log(person.name); //rose

什么鬼？脑袋都懵了。如果说真是按引用传递的话，为什么没有输出”andy”呢？我们一个一个来解释。

第一个例子结果为什么是rose呢？

前面我们说过引用类型的复制，那么这个参数传递的过程肯定是一个浅拷贝的过程，也就是说会把指向对象person的”指针“（其实就是变量名，js中没有指针）赋值给函数的形参obj，那么此时修改形参肯定会引起实参person的变化，因此结果就是”rose”

那为什么第二个例子的结果是还是rose呢？

既然形参obj是指向实参person的指针，那为什么我们后面将name属性置为”andy”时，person没有改变呢？是因为我们new了一个新的对象，此时在堆中会为新对象obj分配一块空间，形参obj将指向这个新对象，而不再指向实参person对象，所以此时改变obj不会对person造成任何影响。函数中的形参就是一个局部变量，在函数执行之后就会被销毁，因此，再也找不到它了
