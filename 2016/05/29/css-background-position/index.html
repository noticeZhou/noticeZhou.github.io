css background-position

  • CSS

今天重新看了一下background-position这个属性。以前在用的时候，没有想那么多，就觉得它就是一个背景图片的定位，没有什么值得注意。也不是说很难，只不过呢，如果不了解透彻了，用的时候就有点懵懵懂懂的，说不出个所以然来，我最不喜欢这种感觉了。

首先呢，background-position这个属性有三种类型的属性值，分别是

  • 数值
  • 关键词
  • 百分比

数值这个很简单，是第四象限坐标的两个值，第一个值是横坐标的值，第二个坐标是纵坐标的值，默认为0.比如： 

bckground-position: 10px 10px;
//这个表示的就是背景图片距离容器左上角上方和左方的距离各位10px

background-position: 20px;
//这个就表示图片距离容器上方20px

百分比就相对复杂一点，属性值由一个或者两个构成，不写默认为0。它的复杂之处在于，它的百分比不是简单的相对于容器的百分比或者相对于图片的百分比。而是，图片的百分比处与容器的百分比处对齐，举例子来说： 

background-position: 50%;
//这就意味着图片的中线与容器的中线重合

background-position:100%;
//这个表示图片的最右端与容器的最右端对齐

关键词有，top,bottom,left,right,center。默认为center，这个是没有顺序要求的，比方说，top left跟left top的效果是一样的，

这三种类型的值是可以混合用的。实际上，background-position可以拆分为background-position-x和background-position-y。所以”50% 50px”，”top 50%”这些值是可以使用的，但是”50px 50px 50px”，”50% 40px 40px”这些值是无效的，因为无法确定x,y值浏览器渲染的时候会把它重置为”0 0”。
但是关键词联合数值或百分比是可以使用的，举个栗子： 

background-position: top 50% left 40px;
//这个其实可以拆解为background-position-x: left 40px;
                  background-position-y: top 50%;
//也就是图片距离容器左边40px，图片高度50%与容器高度50%处重合

css sprites

说完了background-position，不可避免的要说一下css sprites，也就是所谓的雪碧图。什么是雪碧图呢，我们知道浏览器在解析html文件的时候每遇到一个url都会像服务器发起一次http请求，如果网站图片多的话，那么就会发起很多次的请求，网站性能就会减弱。那么一个想法就出来了，可不可以把这些图片合并成一张大的图片呢？这样就只要发起一次http请求就ok了。这就是雪碧图的基本思想。

那么我们将每张图片放到了一张图上面，用的时候怎么找到它呢？这时候就要用到background-position。

background-poaition它是一个相对的属性，属性值为数值的时候，就是图片的左上角相对容器的左上角。就像上面这张图所示，假设所需要的图的左上角在雪碧图的坐标为(200,90),大小为40*40.那么background-position:-200px -90px就可以定位到所需要的图，再将width,height设为40px就可以将期望的图片取出来了。
